Hacking Methodology

সারাংশ

প্রত্যেক পেনিট্রেশন টেস্টার এবং ব্যক্তির নিজস্ব চিন্তাভাবনা, অভিজ্ঞতা এবং টেস্টিং অ্যাপ্রোচ থাকে। নিচে আমার ব্যক্তিগত সিকিউরিটি টেস্টিং মেথডোলজি এবং ওয়ে সংক্ষেপে দেখানো হলো — যা আমার প্র্যাকটিস, লার্নিং এবং রিয়েল-ওয়ার্ল্ড অভিজ্ঞতার উপর ভিত্তি করে তৈরি। এটা OWASP-এর মতো স্ট্যান্ডার্ডের সাথে মিল রেখে কাস্টমাইজ করা, কিন্তু আমার নিজস্ব ফ্লো এবং টুলসের উপর ফোকাস করে।

আমার প্রধান ফোকাস: OWASP (ওয়েব ও API সিকিউরিটি), PTES (স্ট্রাকচার্ড এক্সিকিউশন), NIST SP 800-115 (টেকনিক্যাল গাইডলাইন)

ধাপসমূহ

১. লক্ষ্য নির্বাচন এবং স্ক্যানিং

  • স্কোপ নির্ধারণ ও লিখিত অনুমতি গ্রহণ
  • Nmap – পোর্ট স্ক্যান, সার্ভিস ও ভার্সন ডিটেকশন
  • Nessus – ভালনারেবিলিটি স্ক্যান
  • Acunetix – ওয়েব অ্যাপ্লিকেশন স্ক্যানিং

২. রিকনয়েসান্স ও গোপন রিসোর্স অনুসন্ধান

  • পাবলিক সোর্স ও স্ক্যান রেজাল্ট থেকে ইন্টেলিজেন্স সংগ্রহ
  • Dirbuster / Gobuster / FFuF – হিডেন ডিরেক্টরি ও ফাইল ব্রুটফোর্স
  • সেনসিটিভ ফাইল (PDF, config, backup ইত্যাদি) শনাক্তকরণ

৩. পোর্ট ও সার্ভিস এনুমারেশন

  • খোলা পোর্টের বিস্তারিত ফিঙ্গারপ্রিন্টিং
  • দুর্বল কনফিগারেশন, ডিফল্ট ক্রেডেনশিয়াল চেক

৪. ওয়েব অ্যাপ্লিকেশন টেস্টিং (OWASP Top 10)

  • ম্যানুয়াল টেস্টিং: XSS, SQLi, IDOR, Auth bypass ইত্যাদি
  • অটোমেটেড: Acunetix, Burp Suite, OWASP ZAP
  • প্রায়োরিটাইজেশন ও প্রুফ-অফ-কনসেপ্ট এক্সপ্লয়ট

৪.১ API পেনিট্রেশন টেস্টিং

  • OWASP API Security Top 10 চেক: Broken Object Level Authorization, Excessive Data Exposure ইত্যাদি
  • টুলস: Postman, Burp Suite দিয়ে API এন্ডপয়েন্ট টেস্টিং (রেট লিমিট, অথেনটিকেশন বাইপাস, ইনজেকশন)
  • ম্যানুয়াল এবং অটোমেটেড স্ক্যান: API কী, টোকেন, গ্রাফকিউএল/REST স্পেসিফিক ভালনারেবিলিটি

৫. এক্সপ্লয়টেশন ও অ্যাক্সেস গেইন

  • চিহ্নিত দুর্বলতা এক্সপ্লয়ট (Metasploit / কাস্টম PoC)
  • Exploit chaining ও pivoting

৬. পোস্ট-এক্সপ্লয়টেশন ও পার্সিস্টেন্স

  • ব্যাকডোর / scheduled tasks / credential dumping
  • ল্যাটারাল মুভমেন্ট ও ডোমেইন ডমিনেশন (যদি প্রযোজ্য হয়)
  • ব্যাকআপ অ্যাক্সেস পথ তৈরি

Summary

Every penetration tester and individual has their own unique thinking, experience, and testing approach. Below is a short overview of my personal security testing methodology and way — built from my practice, learning, and real-world experience. It is customized while aligning with standards like OWASP, but focused on my own flow and tools.

My Core Focus: OWASP (Web & API Security), PTES (Structured Execution), NIST SP 800-115 (Technical Guidelines)

Steps

1. Target Selection and Scanning

  • Scope determination and written permission acquisition
  • Nmap – Port scan, service and version detection
  • Nessus – Vulnerability scan
  • Acunetix – Web application scanning

2. Reconnaissance and Hidden Resource Discovery

  • Gather intelligence from public sources and scan results
  • Dirbuster / Gobuster / FFuF – Hidden directory and file brute-force
  • Sensitive file detection (PDF, config, backups, etc.)

3. Port and Service Enumeration

  • Detailed fingerprinting of open ports
  • Weak configuration and default credential checks

4. Web Application Testing (OWASP Top 10)

  • Manual testing: XSS, SQLi, IDOR, Auth bypass, etc.
  • Automated: Acunetix, Burp Suite, OWASP ZAP
  • Prioritization and proof-of-concept exploits

4.1 API Penetration Testing

  • OWASP API Security Top 10 checks: Broken Object Level Authorization, Excessive Data Exposure, etc.
  • Tools: Postman, Burp Suite for API endpoint testing (rate limiting, authentication bypass, injection)
  • Manual and automated scans: API keys, tokens, GraphQL/REST specific vulnerabilities

5. Exploitation and Access Gain

  • Exploit identified vulnerabilities (Metasploit / custom PoC)
  • Exploit chaining and pivoting

6. Post-Exploitation and Persistence

  • Backdoors / scheduled tasks / credential dumping
  • Lateral movement and domain domination (if applicable)
  • Backup access path creation
